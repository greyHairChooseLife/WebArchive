# 04. 라이브 탭 뷰

상태: 진행 중

## 범위

-   `lib/tabs.ts`: `chrome.tabs.query()` 결과를 윈도우별/도메인별/시간대별로 그룹핑 (D23 버킷)
-   상단 그룹핑 토글(윈도우/도메인/시간대), 한 번에 한 관점
-   탭 카드: favicon + title + 도메인 + "마지막 본 지" 표시, 체크박스 다중 선택
-   가져오기: 선택 탭 → 대상 group 지정(기존 선택 또는 새 group) → note로 저장
-   일괄 가져오기: "현재 윈도우 전체 / 모든 윈도우 전체" 버튼
-   `App.tsx`의 탭 뷰 placeholder를 실제 컴포넌트로 교체

## 결정 참조

-   features.md 1번(라이브 탭 멀티 관점 뷰), 2번(탭 가져오기 → note 저장)
-   D19 (라이브 탭은 저장 대상 아님, 가져오기 시에만 note 변환)
-   D20 (inbox 없음, 가져올 때 group 필수 지정)
-   D23 (시간대 그룹핑 버킷, 경계 상수는 `lib/tabs.ts` 한 곳)

## 확정 사항 (이번 단계에서 새로 정함)

-   탭 조회는 패널 진입/탭뷰 전환 시 1회성 `chrome.tabs.query()`. `onUpdated`/`onRemoved` 구독 없음(D22 단순함 우선, architecture.md 미해결 항목 중 "탭 뷰 갱신 방식"을 1회성으로 해소).
-   가져오기 대상 group 지정은 드롭다운(기존 group) + "새 group" 옵션 선택 시 `prompt()`로 이름 입력 → `addGroup` 호출. 커스텀 모달 없음.
-   개별 가져오기(체크박스 선택 후 가져오기)와 일괄 가져오기(현재 윈도우 전체 / 모든 윈도우 전체) 버튼 둘 다 포함. 일괄 버튼은 해당 탭 전체를 선택 상태로 만든 뒤 동일한 가져오기 흐름 재사용.
